## 3. 네이버 OAuth2 설정 (진행중)

### 서버 측 설정
- systemd 환경변수 추가 필요:
  - `NAVER_CLIENT_ID`
  - `NAVER_CLIENT_SECRET`
- Spring Security에 네이버 프로바이더 자동 등록됨

### 프로바이더 등록 진행중
- 네이버 개발자 센터: https://developers.naver.com
- 애플리케이션 등록 필요
- Redirect URI: `https://sunghoonyk.duckdns.org/login/oauth2/code/naver`

### 네이버 등록 메뉴얼
1. https://developers.naver.com 접속
2. **애플리케이션 등록** 클릭
3. 입력:
   - 애플리케이션 이름: `SH Platform`
   - 사용 API: **네이버 로그인** 체크
   - 서비스 URL: `https://sunghoonyk.duckdns.org`
   - Callback URL: `https://sunghoonyk.duckdns.org/login/oauth2/code/naver`
4. **네이버 로그인** 메뉴 → **API 등록** → **네이버 로그인** 선택
5. **Client ID**, **Client Secret** 확인

## 5. 문제 해결 기록

### 문제 1: OAuth2 엔드포인트 404 에러
- 원인: KAKAO_CLIENT_ID 환경변수 미설정
- 해결: systemd 서비스에 환경변수 추가

### 문제 2: nginx에서 OAuth2 경로 미프록시
- 원인: `/oauth2/`, `/login/` 경로가 nginx에 미설정
- 해결: nginx 설정에 proxy location 추가

### 문제 3: 카카오 KOE006 에러
- 원인: Redirect URI 미등록
- 해결: 카카오 개발자 콘솔에서 Redirect URI 등록
