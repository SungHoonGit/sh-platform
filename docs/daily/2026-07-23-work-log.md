## 1. Platform CSS @source 경로 수정

`platform/frontend/src/index.css`의 `@source` 경로가 `../../../../` (4단계向上)이었으나, platform은 `platform/frontend/src/`에서 시작하므로 `../../../` (3단계向上)이 맞았음.

- **원인**: scraper는 `modules/scraper/frontend/src/` (depth 4)이라 `../../../../common/frontend/src` 정상
- **수정**: platform은 `platform/frontend/src/` (depth 3) → `../../../common/frontend/src`
- **결과**: platform CSS에 `bg-blue-600`, `h-14`, `border-slate-700`, `tracking-tight` 등 CommonHeader 클래스 23개 추가 포함
- **커밋**: `90d98fc`

## 2. CI/CD 안정화

### 2.1 fuser 미설치 문제 해결
- **문제**: 서버에 `psmisc` 패키지 미설치 → `fuser -k`가 `2>/dev/null`로 silent fail
- **증상**: 7/21부터 살아있던 좀비 프로세스(PID 1704230, 1704231)가 포트 8080/8081 점유 → 새 auth "Port already in use" 실패
- **해결**: `sudo apt-get install -y psmisc` + CI 워크플로우에서 자동 설치
- **커밋**: `0d1ceaa`

### 2.2 nginx 설정 중복 제거
- **문제**: `fix-nginx.py`가 매번 실행될 때마다 중복 location 추가 → CI nginx 테스트 실패
- **해결**: `nginx/sh-platform.conf`를 git에 저장, CI에서 직접 복사 방식으로 전환
- **커밋**: `9308d42`, `17b1bbe`

### 2.3 서비스 시작 타임아웃

## 3. AuthGuard 공통 컴포넌트 (e096818)

`/scraper-ui/` 비로그인 상태에서 헤더 네비게이션 클릭 시 하얀 화면 → 로그인 플로우 미작동 문제 해결.

### 문제 원인
1. CommonHeader `<a href="/platform">` → 전체 페이지 이동
2. Platform App.tsx의 `<Navigate to="/">`가 `<BrowserRouter>` **바깥**에서 렌더링 → React Router 에러 → 하얀 화면
3. Scraper에는 auth guard 없음 → 로그인 안 해도 모든 페이지 접근 가능

### 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `common/frontend/src/components/AuthGuard.tsx` | **신규** — `useAuth()`로 세션 확인, 없으면 `"/"` 로 리다이렉트 |
| `common/frontend/src/index.ts` | `AuthGuard` export 추가 |
| `common/frontend/src/components/CommonHeader.tsx` | 미로그인 시 네비게이션 링크 → `"/"` (로그인)으로 |
| `modules/scraper/frontend/src/components/Layout.tsx` | `<AuthGuard>` 래핑 |
| `platform/frontend/src/layouts/PlatformLayout.tsx` | `<AuthGuard>` 래핑 |
| `platform/frontend/src/App.tsx` | `<Navigate>`가 Router 바깥에 있던 버그 제거, AuthGuard로 교체 |

### 동작
- `/scraper-ui/`, `/platform/` 비로그인 접근 → AuthGuard가 `/` (로그인 페이지)로 리다이렉트
- 헤더의 "대시보드", "이력서" 등 클릭 → 로그인 안 되어 있으면 모두 `/`로
- 로그인 후 → 정상 접근
- `common/frontend`의 AuthGuard 하나로 **모든 SPA에 일괄 적용**
