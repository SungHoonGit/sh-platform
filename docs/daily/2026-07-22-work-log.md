## 2. nginx sites-enabled vs sites-available 버그 (6abe757)

### 원인

`/etc/nginx/sites-available/sh-platform`에 `/scraper-ui/` location을 추가했지만, nginx가 실제로 읽는 파일은 `/etc/nginx/sites-enabled/sh-platform`이었음. `sites-enabled`에는 `/scraper-ui/` location 자체가 없었고 모든 요청이 `location /` (auth frontend)로 fallthrough.

### 해결

- `fix-nginx.py`의 `CONFIG`를 `sites-available` → `sites-enabled`로 변경
- `alias` + `error_page 404` 방식으로 파일 서빙 (nginx 1.24.0 alias+try_files 버그 회피)
- `= /scraper-ui` → 301 → `/scraper-ui/` 리다이렉트 추가

## 3. Tailwind v4 symlink 콘텐츠 감지 문제 (e7981c5)

### 원인

Tailwind v4의 자동 콘텐츠 감지가 `@common` 심볼릭 링크를 따라가지 못함. scraper CSS에 `bg-slate-900`, `text-white`, `h-14` 등 CommonHeader 핵심 클래스가 미포함.

### 해결

`modules/scraper/frontend/src/index.css`에 추가:

```css
@import "tailwindcss";
@source "../../../../common/frontend/src";
```
