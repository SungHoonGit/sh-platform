# 관리자 플랫폼 프레임 설계

## 1. 목적

- 시스템 관리자를 위한 플랫폼 프레임
- 사용자 계정 관리, 테넌트 관리, 시스템 모니터링
- 각 모듈(스크래퍼, 이력서, 포트폴리오) 진입점 제공

## 4. API 연동

### 4.1 사용 API (Auth 백엔드)

| 기능 | Method | URL | 설명 |
|------|--------|-----|------|
| 사용자 목록 | GET | `/api/v1/admin/users` | 페이징, 검색, 필터 |
| 사용자 상세 | GET | `/api/v1/admin/users/{id}` | 상세 정보 |
| 사용자 수정 | PUT | `/api/v1/admin/users/{id}` | 역할, 상태 변경 |
| 사용자 삭제 | DELETE | `/api/v1/admin/users/{id}` | soft delete |
| 테넌트 목록 | GET | `/api/v1/admin/tenants` | 페이징, 검색, 필터 |
| 테넌트 상세 | GET | `/api/v1/admin/tenants/{id}` | 상세 정보 + 멤버 |
| 테넌트 수정 | PUT | `/api/v1/admin/tenants/{id}` | 플랜, 상태 변경 |
| 테넌트 삭제 | DELETE | `/api/v1/admin/tenants/{id}` | soft delete |
| 멤버 역할 변경 | PUT | `/api/v1/admin/tenants/{id}/members/{memberId}` | 역할 변경 |
| 멤버 초대 | POST | `/api/v1/admin/tenants/{id}/members` | 이메일 초대 |
| 멤버 제거 | DELETE | `/api_v1/admin/tenants/{id}/members/{memberId}` | 멤버 제거 |
| 대시보드 통계 | GET | `/api/v1/admin/stats` | 사용자, 테넌트 수 등 |
| 시스템 설정 | GET/PUT | `/api/v1/admin/settings` | 시스템 설정 |

### 4.2 신규 API 엔드포인트 (백엔드 구현 필요)


## 6. 인증/인가

### 6.1 JWT 토큰 관리

```typescript
// 로그인 시
localStorage.setItem("accessToken", token);
localStorage.setItem("refreshToken", refreshToken);

// API 호출 시
headers: {
  "Authorization": `Bearer ${accessToken}`
}

// 토큰 만료 시
// → 리프레시 토큰으로 갱신
// → 갱신 실패 시 로그인 페이지로 이동
```

### 6.2 권한 확인

```typescript
// 사용자 역할 확인
const user = useAuthStore(state => state.user);

// ADMIN만 접근 가능한 페이지
if (user?.role !== "ADMIN") {
  return <Navigate to="/platform/dashboard" replace />;
}
```

## 7. 구현 순서

### Phase 1: 백엔드 API (Admin 컨트롤러)
1. AdminController 생성
2. 사용자 CRUD API
3. 테넌트 CRUD API
4. 대시보드 통계 API

### Phase 2: 프론트엔드 공통 컴포넌트
1. Layout, Sidebar, Header
2. DataTable, Modal
3. SearchFilter, StatsCard

### Phase 3: 관리자 페이지
1. 대시보드
2. 사용자 관리
3. 테넌트 관리
4. 설정

### Phase 4: 인증 연동
1. 로그인/로그아웃
2. JWT 토큰 관리
3. 권한 확인

## 8. 참고 사항

- 관리자 페이지는 ADMIN만 접근 가능
- 모든 API는 JWT 인증 필요
- 테넌트 관련 API는 OWNER/ADMIN만 접근 가능
- 사용자 삭제는 soft delete (데이터 보존)
