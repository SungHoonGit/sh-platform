# 다국어 처리 방안

## 1. 전략

Spring Boot `MessageSource` 기반, 클라이언트가 `Accept-Language` 헤더로 언어 전달.

### 현재

```
ko — 한국어 (기본)
en — 영어
```

### 추후 확장 시

```
ja — 일본어 (AI Housing 공고에 일본 부동산 정보 추가 시)
zh — 중국어
```

---

## 2. 파일 구조

```yaml
src/main/resources/
├── messages/
│   ├── messages.properties          # default (영어, fallback)
│   ├── messages_ko.properties       # 한국어
│   └── messages_en.properties       # 영어
├── validation/
│   ├── validation.properties        # default
│   ├── validation_ko.properties     # @Valid 에러 메시지
│   └── validation_en.properties
└── application.yml
```

---

## 3. 설정

```yaml
spring:
  messages:
    basename: messages/messages, messages/validation
    encoding: UTF-8
    fallback-to-system-locale: false
```

```java
@Configuration
public class MessageConfig {
    @Bean
    public MessageSource messageSource() {
        ReloadableResourceBundleMessageSource ms = new ReloadableResourceBundleMessageSource();
        ms.setBasename("classpath:messages/messages");
        ms.setDefaultEncoding("UTF-8");
        ms.setCacheSeconds(3600);
        return ms;
    }

    @Bean
    public LocaleResolver localeResolver() {
        AcceptHeaderLocaleResolver resolver = new AcceptHeaderLocaleResolver();
        resolver.setDefaultLocale(Locale.KOREA);
        return resolver;
    }
}
```

```properties

```

## 5. 사용 예시

### Service에서 메시지 참조

```java
@Service
public class AuthService {

    private final MessageSource messageSource;

    public void signup(SignupRequest request) {
        if (userRepository.existsByEmail(request.email())) {
            throw new BusinessException("auth.signup.email.duplicate");
        }
        // ...
    }
}
```

### GlobalExceptionHandler에서 메시지 변환

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @Autowired
    private MessageSource messageSource;

    @ExceptionHandler(BusinessException.class)
    public ResponseEntity<ApiResponse<Void>> handle(BusinessException e, Locale locale) {
        String message = messageSource.getMessage(e.getCode(), e.getArgs(), locale);
        return ResponseEntity
            .status(e.getHttpStatus())
            .body(ApiResponse.error(e.getCode(), message));
    }
}
```

### 클라이언트 요청 예시

```bash
# 한국어 응답
curl -H "Accept-Language: ko" https://api.sung-hoon.io/api/auth/login

# 영어 응답
curl -H "Accept-Language: en" https://api.sung-hoon.io/api/auth/login
```

## 6. DB 다국어 (추후 확장)

고정 값 (예: 공고 카테고리, 코드성 데이터)은 테이블로 관리:

```sql
CREATE TABLE codes (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    group_code  VARCHAR(50) NOT NULL,       -- CATEGORY, JOB_TYPE
    code        VARCHAR(50) NOT NULL,       -- APT, OFFICETEL
    sort_order  INT DEFAULT 0,
    UNIQUE KEY uk_codes (group_code, code)
);

CREATE TABLE code_messages (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    code_id     BIGINT NOT NULL,
    locale      VARCHAR(10) NOT NULL,       -- ko, en, ja
    message     VARCHAR(200) NOT NULL,
    UNIQUE KEY uk_code_messages (code_id, locale),
    FOREIGN KEY (code_id) REFERENCES codes(id) ON DELETE CASCADE
);
```

조회 예시:

```sql
SELECT cm.message
FROM codes c
JOIN code_messages cm ON cm.code_id = c.id
WHERE c.group_code = 'CATEGORY'
  AND c.code = 'APT'
  AND cm.locale = 'en';
```

## 7. 적용 우선순위

| 단계 | 범위 | 방식 |
|:----:|------|------|
| Phase 2 | API 응답 메시지 | `messages_ko.properties` (한국어만) |
| Phase 3 | + 영문 메시지 | `messages_en.properties` 추가 |
| Phase 4 | + DB 코드성 데이터 | `codes` + `code_messages` 테이블 |
| Phase 5 | + Validation 에러 | `validation_*.properties` |

> Phase 2에서는 **messages_ko.properties**만 작성, 영어는 추후 추가.
> 모든 에러 코드는 `BusinessException`에 code 문자열로 정의 → MessageSource에서 조회.
